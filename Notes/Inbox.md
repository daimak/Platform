# Inbox

## 1. Принципы (черновики, НЕ часть системы)

- P9. Реальность фиксируется в систему сразу  
- P10. Разделение уровней идентичности и структуры  
- P11. Контроль источника истины  
- P12. Разделение статической и динамической информации  
- P13. Dashboard не содержит план создания системы  
- P14. Контроль источника истины (дублировался по смыслу)  
- P15. Черновые принципы вне системы  
- P16. Структурное разделение пространства  
- P17. Единственный источник состояния системы — локальная файловая система  
- P18. Dashboard всегда содержит только выбор действия  
- P19. Один концепт — один термин  
- P20. Принципы существуют только после фиксации в Obsidian  
- P21. Черновики должны существовать отдельно от системы  

---

## 2. Возможные модули системы

- Migration (Узбекистан) — переезд / жизнь / база
- Career — международный доход ($2500+)
- Finance — накопления, бюджет, runway
- Infrastructure — техническая база (VM, cloud, sandbox)

---

## 3. Архитектурные идеи

- разделение системы на:
  - Core (ядро)
  - Entry (вход: README / Dashboard)
  - Context (Genesis / Session notes)
  - Logs (Memos / CHANGESET)
  - Inbox (черновики)

- Dashboard = состояние системы (динамика)
- README = карта системы (статика)

---

## 4. Процесс работы системы

- короткие итерации (P7 расширение идеи)
- шаг → фиксация → следующий шаг
- любое изменение фиксируется через CHANGESET
- система развивается через маленькие проверяемые шаги

---

## 5. Управление ИИ

- взаимодействие через короткие шаги
- без длинных цепочек действий
- обязательная проверка после каждого шага
- система должна оставаться контролируемой

---

## 6. Идеи инфраструктуры

- виртуальная машина / sandbox для экспериментов
- cloud окружение (бесплатные или минимальные)
- резервное хранение на ноутбуке жены
- защита от потери данных (fallback storage)

---

## 7. Общая концепция Платформы

- Платформа = операционная система жизни
- цель: эволюционирующая система, а не статичный план
- устойчивость важнее фиксированного плана
- система должна адаптироваться к смене профессий и технологий (включая трансформацию IT)
